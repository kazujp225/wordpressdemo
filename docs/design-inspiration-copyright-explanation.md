# デザインインスピレーション機能と著作権について

## 結論

このシステムは**参照画像の「スタイル」や「感性」を抽出して適用するもの**であり、**画像や具体的表現のコピーではありません**。著作権法上、「スタイル」「アイデア」「配色」などの抽象的要素は保護対象外であり、法的リスクは極めて低いと考えられます。

---

## 1. 技術的に何をしているか

### 抽出するもの（抽象的要素のみ）

| 抽出要素 | 具体例 | 著作権保護 |
|----------|--------|-----------|
| カラーパレット | `#1A1A2E`, `#E94560` など4色 | **保護されない** |
| タイポグラフィ | "Modern, Bold" などのスタイル記述 | **保護されない** |
| レイアウト密度 | "Spacious", "Compact" など | **保護されない** |
| 雰囲気（Vibe） | "Luxury, Minimalist, Corporate" | **保護されない** |
| スタイル説明 | "Dark mode with neon accents" | **保護されない** |

### 抽出しないもの（コピーしない）

- 実際の画像・写真
- テキストコンテンツ
- 具体的なイラスト・グラフィック
- ロゴ・商標
- レイアウトの具体的な座標・配置


### デザイン統一API（`src/app/api/ai/design-unify/route.ts`）

```typescript
【やってはいけないこと】
- 参照画像の内容をそのままコピーする
- レイアウトを崩す
```

### 分析API（`src/app/api/ai/analyze-design/route.ts`）

分析対象を以下に限定：
- 色相（hue）と彩度（saturation）
- 背景の質感（マット/グロス/粒状）
- 光源方向とコントラスト

**画像の具体的内容は分析・抽出しない**

## 3. 著作権法の観点から

### 著作権で保護されるもの/されないもの

| 保護される | 保護されない |
|-----------|-------------|
| 具体的な創作的表現 | アイデア・コンセプト |
| 写真・イラストそのもの | 配色・色の組み合わせ |
| 独創的なレイアウト設計 | 「スタイル」「雰囲気」 |
| 文章・コピー | 一般的なレイアウトパターン |

### 判例・法的見解

**アイデア・表現二分論**（著作権法の基本原則）：
> 著作権は「具体的な表現」を保護するものであり、その背後にある「アイデア」「スタイル」「手法」は保護対象外

**色彩・配色について**：
> 単純な色の組み合わせや配色パターンは著作物として認められない（東京地裁平成13年など）

**デザインスタイルについて**：
> 「ミニマリスト」「ダークモード」「コーポレート」などのスタイル・トレンドは万人が使用可能

## 4. 類似サービスとの比較

### 本システムがやっていること

```
参照画像 → AI分析 → 抽象的スタイル情報抽出 → 新規画像生成
          （色、雰囲気のみ）       （別のコンテンツで）
```

### 著作権侵害となりうる例

```
参照画像 → そのままコピー → トレース or 切り貼り
```

**本システムは後者ではない**

## 5. 具体的な実装による保護

### Style Anchor方式

1. 参照画像から**色・質感・照明方向のみ**を抽出
2. これを「スタイル基準」として保持
3. 各セクションで**新規コンテンツを生成**（コピーではない）

### セーフガード機構

```typescript
// マスク外の領域を元画像で上書きして、確実に維持する
const finalBuffer = await applyMaskSafeguard(
    originalBuffer,    // 元画像
    generatedBuffer,   // 生成画像
    ...
);
```

編集時も、指定領域外は元画像を完全保持し、不要な変更を防止。

## 6. 想定Q&A

### Q: 参照画像のデザインに「似ている」と言われたら？

A: 「似ている」こと自体は著作権侵害ではありません。著作権侵害が成立するには：
1. **依拠性**：実際にその著作物を参考にしたか
2. **類似性**：具体的な創作的表現が実質的に同一か

本システムは「スタイル」のみを参照し、具体的表現は新規生成しているため、法的な「類似性」は認められにくい。

### Q: 有名なサイトのスクリーンショットを参照に使っても大丈夫？

A: 抽出するのは「カラーパレット」「雰囲気」などの抽象的要素のみであり、具体的なデザイン要素（ロゴ、画像、レイアウトの詳細など）はコピーされません。ただし、ブランドアイデンティティを構成する特徴的な色の組み合わせなどは、商標権・不正競争防止法の観点から慎重に扱うことを推奨します。

### Q: これはAIによる「学習」と同じでは？

A: 本システムは：
- 参照画像を学習データとして保存しない
- モデルの重みを更新しない
- セッション中のみ一時的に分析結果を使用

生成AIの学習問題とは異なる、「プロンプト内での参照指示」に過ぎません。

## 7. 推奨事項

### ユーザーへの案内として

1. **自社・クライアントのデザインを参照に使う**：最もリスクが低い
2. **汎用的なテンプレートを参照に使う**：問題なし
3. **競合他社のサイトを参照に使う**：技術的には可能だが、不正競争防止法の観点から推奨しない
4. **著名ブランドのサイトを参照に使う**：色やロゴがブランドアイデンティティを構成する場合は避ける

### 免責事項（利用規約への記載推奨）

> 本サービスのデザインインスピレーション機能は、参照画像から抽象的なスタイル情報（配色、雰囲気など）のみを抽出し、新規コンテンツの生成に活用するものです。参照画像の著作権者の権利を侵害する目的での使用は禁止します。生成されたコンテンツに関する法的責任はユーザーが負うものとします。

---

## まとめ

| 観点 | 本システムの実装 | 法的リスク |
|------|-----------------|-----------|
| 画像のコピー | **しない**（プロンプトで禁止） | なし |
| スタイル・色の参照 | **する**（抽象的要素のみ） | なし（保護対象外） |
| 具体的レイアウトのコピー | **しない** | なし |
| ブランド要素の使用 | **禁止**（プロンプトで明記） | なし |

**結論**：本システムは著作権法上の「アイデア・表現二分論」に基づき、保護対象外の抽象的スタイル情報のみを活用しており、適法なサービスです。
